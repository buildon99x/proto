# docs/kb/concept-planning — 컨셉 기획 골든셋 (Golden Set)

> **목적:** 컨셉 기획(`docs/concepts/**`)의 **품질을 끌어올리는 지식 자산**. 실존 게임을 *역기획(reverse-engineer)*하고 공개된 게임디자인 페이퍼를 조사해, ① **인사이트**를 도출하고 ② 재사용 가능한 **컨셉 기획 템플릿**을 만들고 ③ 성공/실패 + 현 시장 주류 **골든셋(실 사례 117종, 10대 장르 각 10종 이상)**을 구축한다.
> **위상:** 지식베이스(KB). 코드 변경 없음. `docs/concepts/`(컨셉 발굴 실무)가 *생산물*이라면 이 폴더는 그 실무를 **교정·채점·영감을 주는 기준선(reference)**이다.
> **언어:** 한국어(`docs/concepts/**`·`docs/design/**`·`.context/kb/**`와 동일 규약).
> **갱신일:** 2026-06-14.

## 왜 골든셋인가

컨셉 기획의 실패 양상은 두 가지다 — **(a)** 한눈에 상상이 안 되는 모호한 기획서, **(b)** 멋져 보이지만 시장에서 죽는 컨셉. 골든셋은 이 둘을 동시에 막는다.

- **(a)에 대해:** 117개 실 사례가 "충분히 설명되고 상상되는 기획서"의 **구체적 본보기**가 된다. 새 컨셉 기획서를 작성하면 골든셋의 동급 사례와 나란히 놓고 *"이 사례만큼 선명한가?"*를 물을 수 있다(장르별 비교는 §05 인덱스).
- **(b)에 대해:** 성공작과 실패작을 **같은 템플릿으로 역기획**해 비교하면, "무엇이 컨셉 단계에서 성패를 갈랐는가"가 **반복 패턴(§00 인사이트)**으로 드러난다. 그 패턴은 채점 루브릭(§04)의 레드플래그가 된다.

> **핵심 명제(§00에서 도출):** *상업적 성공은 컨셉 단계에서 이미 상당 부분 결정된다.* 한 줄 후크의 선명함, "X+Y" 차별화의 정당성, 30초 데모 가독성, 라이브서비스 추종(me-too)의 회피 — 이것들은 모두 **컨셉 문서에서 검사 가능**하다.

## 자산화 — 기획할 때 활용하는 법

이 KB는 *읽는 문서*를 넘어 **활성 자산**이다. 세 가지 진입점:

1. **스킬(권장):** `/concept-design` — 새 컨셉을 §01 템플릿으로 작성하고, 동급 골든셋과 비교하고, §04 루브릭·레드플래그로 채점하는 전 과정을 구동. 기존 컨셉 채점·골든셋 질의도 지원.
2. **도구(CLI):** 빠른 질의·점검:
   ```bash
   python3 docs/kb/concept-planning/tools/score_concept.py stats                 # 장르 ≥10 검증
   python3 docs/kb/concept-planning/tools/score_concept.py compare --genre G1 --verdict success
   python3 docs/kb/concept-planning/tools/score_concept.py find 포커
   python3 docs/kb/concept-planning/tools/score_concept.py lint docs/concepts/03-my-concept.md
   ```
3. **JSON 소스:** `data/golden-set.json`을 직접 필터/가공(장르·판정·X+Y·인사이트). 마크다운 문서(02/03/06)와 항상 일치시킨다 — *사례를 더하거나 고치면 JSON과 해당 마크다운을 함께 갱신*.

1. **새 컨셉을 쓰기 전:** §00 인사이트와 §02/§03에서 **인접 장르 사례**를 읽는다(영감 + 함정 인지).
2. **컨셉을 쓸 때:** §01 템플릿을 채운다. 각 섹션의 *품질 바*를 통과할 때까지.
3. **컨셉을 쓴 뒤:** §04 루브릭으로 자가 채점하고, **동급 골든셋 사례와 나란히** 두어 "이만큼 선명한가/정당한가"를 검사한다. 레드플래그가 켜지면 §03의 해당 실패 사례로 돌아간다.

> `docs/concepts/00–02`는 이 골든셋의 **첫 소비자**다. 그 문서들의 6기준 채점과 "X+덱빌더" 차별화 명제는 여기 §00·§04에서 일반화·강화된다.
