# Spec

> 기획 단계 스펙. 게임 디자인의 전체 스코프를 규정하고, 구현 착수 시 이 문서가 빌드 기준이 된다. 상세 수치·표는 `docs/design/*`로 분할한다.

## 1. 산출물 범위 (현 단계)

- **이번 산출물 = 기획서**: `brief.md` / `spec.md` / `eval.md` + `docs/concept.md`(13섹션 컨셉 기획서) + `docs/design/*`(메커닉·스테이지·캐릭터·메타·UX 플로우).
- **구현 미포함**: 앱 코드·에셋·런처 레지스트리 등록·빌드는 후속 작업. 따라서 현재 `pnpm sync:registry` / `pnpm build:vercel`는 **실행하지 않는다**(빈 앱 등록으로 런처 빌드가 깨지는 것을 방지). 구현 착수 조건은 §8에 명시.

## 2. 카피 베이스 → 드래곤 매핑 (메커닉 정체성)

도돈파치의 메커닉 축을 1:1로 계승하고 테마만 교체한다. 상세는 `docs/design/mechanics.md`.

| 도돈파치 원작 | 본작 |
|---|---|
| 탭=샷(확산) / 홀드=레이저(집중·자기 감속) | 비늘 산탄(샷) / 집중 용염 빔(레이저, 홀드 시 감속=정밀 모드) |
| 체인 콤보(격파 간격 타이머) | **연환(連環) 게이지** — 연속 격파로 유지, 끊기면 배수 리셋 |
| 봄(전체 소거+무적) | **포효(절멸의 숨결)** — 화면 탄막 소거 + 짧은 무적, 재고 제한 |
| 하이퍼(게이지 충전·강화·탄소거) | **각성/광룡화** — 충전 후 발동, 화력 강화 + 발동 시 탄막 소거 |
| 숨은 벌(蜂) 순서 수집 보너스 | **숨은 용비늘/용알** — 지형 파괴로 출현, 순서 수집 시 보너스 + 메타 재화 |
| 적탄 소거 → 점수 아이템화 | 격파·각성 시 적탄이 **정수(精髓) 비늘**로 전환되어 흡수 |
| 2주차(2-loop) + 진 최종보스(히바치) | **심연 회랑(2주차)** + 진 최종보스 **시원의 고룡 '아카식'** |

## 3. 컨트롤

- **이동**: 키보드 방향키 / 좌스틱 / 모바일 가상 패드(드래그 추종).
- **샷(탭/연사)**: 비늘 산탄. 광역·약위력. 자동 연사 옵션 제공.
- **레이저(홀드)**: 집중 용염 빔. 고위력·관통. **홀드 중 이동속도 감소(정밀 회피 모드)** — 도돈파치 레이저 감속 계승.
- **봄(포효)**: 화면 소거 + 무적 프레임. 재고 제한. Novice는 피격 시 **오토봄** 자동 발동.
- **각성(광룡화)**: 게이지 만충 시 수동 발동. 강화 + 발동 순간 탄막 소거.
- 모바일: 샷 자동 + 빔/봄 버튼 2개, 한 손 이동 — 입력 단순화.

## 4. 핵심 규칙 (코어 시스템)

- **연환 스코어링**: 적 격파 시 연환 카운터 +1, 격파 간격이 타이머 내면 유지. 카운터가 점수 배수를 곱연산으로 끌어올린다. 끊기면 배수 리셋. (공식·수치 → `mechanics.md`)
- **샷/레이저 역할 분담**: 샷은 잡몹 광역 처리(연환 유지에 유리), 레이저는 보스·경장갑 고속 격파 + 감속 정밀 회피. 상황별 전환이 코어 의사결정.
- **봄·각성 자원 관리**: 봄을 죽음 회피로 쓸지 점수 보존(노봄 보너스)으로 아낄지, 각성을 언제 터뜨릴지가 트레이드오프.
- **숨은 비늘 수집**: 스테이지 곳곳 지형/오브젝트 파괴로 숨은 용비늘 출현 → **정해진 순서**로 먹으면 누적 보너스(원작 벌 수집 계승). 메타 재화로도 환산.
- **피격/죽음**: 피격 시 잔기 감소(그레이즈/판정은 도돈파치식 소형 히트박스). 잔기 0 → 컨티뉴 선택 or 게임오버.
- **2주차(심연)**: 1주차를 일정 조건(예: 노컨티뉴/일정 점수 등)으로 클리어 시 심연 회랑 진입 → 진 최종보스 '아카식' 해금.

## 5. 콘텐츠 스코프

- **스테이지**: 6스테이지 캠페인(각 ~5–6분, 미드보스+보스). 2주차=심연(고밀도 리믹스)+진보스. 상세 → `docs/design/stages-bosses.md`.
- **플레이어블 드래곤 4종**: 화룡 이그니스(밸런스)·빙룡 글라키에스(고화력 저속)·뇌룡 포보스(고기동 유도)·(해금)암흑룡 녹스(고배수 하이리스크). 각 샷/레이저 성향 분담. 상세 → `docs/design/characters.md`.
- **난이도**: Novice(오토봄 입문) → Original(기본) → Maniac(고밀도) → Ultra/심연(2주차 해금).

## 6. 메타 성장 / 경제

- **재화 = 용비늘(Scales)**: 런의 점수·숨은 수집량·클리어 보너스로 획득.
- **허브 '용소(龍巢)'**: 신규 드래곤 해금 / 드래곤별 영구 업글(화력·봄 소지수·각성 효율·시작 잔기·옵션 추가) / 난이도 해금 / 연습 모드(스테이지 셀렉트·보스 러시) / 도전과제→스킨·보스 도감·뮤직룸.
- **가드레일**: 무 MTX, **발견형 보상**. 메타 업글은 실력 비중을 해치지 않게 폭 절제(P2W 방지). 상세 표 → `docs/design/meta-progression.md`.

## 7. 화면·상태 (E2E UX 플로우)

부팅 → 타이틀 → 드래곤 선택 → 난이도 선택 → 인게임(HUD/일시정지) → 사망/컨티뉴/게임오버 → 리절트(점수·용비늘 정산) → 용소 허브(해금·업글) → 연습/도감/옵션 → 재도전. 모든 화면 구성요소·전이 조건·되돌아가기 경로는 `docs/design/ux-flow.md`에 정의(막다른 화면 없음).

## 8. 구현 메모 (후속 작업 — 착수 시 적용)

- **스택 제안**: Vite + React + TypeScript + Phaser(레포 `altok-dragon-hatchery` 패턴 재사용). 탄막·충돌은 Phaser 캔버스, 텍스트 UI(도감·허브·옵션)는 DOM.
- **결정성 주의**: 점수·연환 곱연산은 정수 수학으로 결정적 처리. 탄막 패턴은 데이터(JSON/DSL)로 분리해 튜닝 가능하게.
- **구현 착수 조건(현재 미수행)**: ① `projects/dragon-danmaku/app` 생성 ② `project.json` 레지스트리 등록 ③ `pnpm sync:registry` ④ `pnpm build:vercel` 검증. 본 기획서 단계에서는 수행하지 않는다.
