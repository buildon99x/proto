# Eval

> 현 단계 산출물은 **기획서**이므로, 수용 기준은 "문서가 게임을 처음부터 끝까지 빈틈없이 정의했는가(E2E 완성도)"로 평가한다. 향후 구현 시의 빌드 수용 기준은 §F에 별도 명시한다.

## E2E 완성도 점검 체크리스트 (기획서 수용 기준)

### A. 전체 플레이 루프 단절 없음 (`docs/design/ux-flow.md` 기준)
- [ ] 부팅 → 타이틀 → 드래곤 선택 → 난이도 선택 → 인게임 → 사망/클리어 → 리절트 → 용소 허브 → 재도전의 **모든 전이가 끊김 없이** 정의됨.
- [ ] 모든 화면·상태 정의: 타이틀, 드래곤 선택, 난이도 선택, 인게임 HUD(잔기/봄/연환/각성/스코어), 일시정지, 컨티뉴, 게임오버, 리절트, 용소 허브, 보스 도감, 뮤직룸, 옵션, 연습/보스 러시.
- [ ] 각 화면에 **되돌아가기/이탈 경로**가 존재한다(막다른 화면 없음).

### B. 시스템 상호연결 폐루프
- [ ] 연환 ↔ 스코어 ↔ 용비늘 ↔ 메타 업글 ↔ 난이도 해금의 수치 흐름이 **한 바퀴 닫힌다**.
- [ ] 무기(샷/레이저)·봄(포효)·각성·숨은 비늘 수집이 **스코어링 공식에 모두 반영**된다.

### C. 콘텐츠 누락 없음
- [ ] 6스테이지 전부 + 각 미드보스/보스 패턴 개요 + 2주차(심연) 진입 조건 + 진 최종보스 '아카식' 해금 조건이 명시됨.
- [ ] 플레이어블 드래곤 4종 × 샷/레이저 성향 전부 정의, 해금 비용 표화.
- [ ] 메타 해금/업글 항목 전부 표로 열거("등등"·빈칸 금지).

### D. 엣지·실패 경로
- [ ] 잔기 0, 봄 0, 연환 끊김, 컨티뉴 사용/거부, 노컨티뉴 조건, 2주차 자격 미달 등 분기 처리가 정의됨.

### E. 기획 품질 (컨셉 루브릭)
- [ ] `docs/concept.md`가 13섹션 전부 + §8 톤앤매너 (a~h) 구체 작성, `docs/kb/concept-planning/04-eval-and-rubric.md` 레드플래그 자가점검 통과.
- [ ] `brief.md`/`spec.md`/`eval.md`가 기존 프로젝트 양식과 일관.
- [ ] 메타·난이도·스테이지 표로 **플레이타임 3–4h+ 산술이 성립**한다.
- [ ] 핵심 검수 질문 통과: "기획서만 읽고 도돈파치를 모르는 사람도 ① 어떤 게임인지 ② 왜 재밌는지 ③ 3–4시간을 어떻게 채우는지가 그려지는가."
- [ ] 레포 빌드 영향 없음(레지스트리 미등록 → 런처 빌드 무영향).

## Verification (현 단계)

- 문서 리뷰: 위 A–E 체크리스트 전 항목 충족 확인.
- 교차 일관성 점검: `concept.md`(§3 톤 ↔ §8 비주얼 ↔ §9 사운드), `mechanics.md` 수치 ↔ `meta-progression.md` 경제 ↔ `stages-bosses.md` 분량이 모순 없는지.
- 레포 무영향 확인: `projects/dragon-danmaku/`는 레지스트리 미등록 → `pnpm build:vercel` 영향 없음.

## F. 프로토타입 빌드 수용 기준 (`app/`)

- [x] 첫 화면에서 드래곤·난이도 선택 후 즉시 1스테이지 진입, 샷/레이저/봄/각성이 모두 동작.
- [x] 연속 격파로 연환 배수가 오르고 끊기면 리셋, 점수에 반영.
- [x] 보스 격파 → 리절트에서 점수·수집량 기반 용비늘 정산 → 허브에서 해금 반영(localStorage 영속).
- [x] 잔기 0 → 컨티뉴/게임오버 분기, 노컨티뉴 시 심연·진보스(아카식) 경로.
- [x] 데스크톱(키보드)·모바일(드래그 이동 + 빔/포효/각성 버튼) 입력.
- **자동 검증(완료)**: `pnpm --filter dragon-danmaku lint`(tsc) / `build`(tsc+vite) / `pnpm sync:registry` / `pnpm validate:projects` / `pnpm build:vercel`.
- **수동 플레이테스트(권장)**: `pnpm --filter dragon-danmaku dev` 후 타이틀→출격→연환 빌드→보스→리절트→허브 해금 1사이클.
