# Spec

## 상태

`brief.md`의 Open Questions가 해소되기 전까지는 초안이다. 방향이 확정되면 화면/상태/데이터
모델을 다시 정의한다.

## 가정한 핵심 상태

- `cash`: 보유 자본 (숫자)
- `machines`: 보유 설비 목록 (id, 개수, 초당 수익, 업그레이드 레벨)
- `clickValue`: 클릭 1회당 획득 자본

## 가정한 핵심 동작

- 클릭 버튼 클릭 시 `cash += clickValue`
- 설비 구매 시 비용만큼 `cash` 차감, 초당 자동 수익 증가
- 틱(예: 1초)마다 보유 설비의 초당 수익 합만큼 `cash` 증가
- 업그레이드 구매 시 클릭 배율 또는 설비 수익 배율 증가

구현 착수 전 브리프의 Open Questions를 확인하고 이 문서를 갱신할 것.
