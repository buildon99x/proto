## 자동 검증
- `pnpm --filter retro-bowling build` — 타입체크(tsc) + Vite 빌드 무오류.
- `pnpm playtest --project retro-bowling` — 헤드리스 크롬으로 타이틀→10프레임→
  게임오버까지 자동 플레이. page/console 에러 0, 스크린샷 8장 캡처.
  시나리오: `tests/e2e/scenario.mjs`.
- 스코어링 단위 검증(수동 스크립트로 확인한 케이스):
  - 퍼펙트 게임(12스트라이크) = 300
  - 올 거터 = 0, 올 9핀 오픈 = 90
  - 올 스페어(5/5…)+마지막 5 = 150
  - 위키 표준 예시 = 133
  - 10프레임 X X X 단독 = 30
  - 보너스 미확정 스트라이크 프레임 누적 = null

## v0.2 밸런스 검증(헤드리스 스트라이크맵)
`scripts`(스크래치) 하니스로 실제 엔진 물리를 헤드리스 구동해 (조준,파워,스핀)→핀 수 측정.
- 정중앙 직구(25, 임의 파워, 스핀0): **스트라이크 안 남**(레인 전 구간).
- 직구 스트라이크율: 약 5% 수준(우연히만).
- 훅 포켓(aim≈30, power≈0.73, spin≈-0.1): 조준 밴드 28~31에서 평균 ~9.7핀,
  7개 조준 중 5개가 스트라이크 → 응집된 포켓.
- 통합 검증(플레이테스트 시나리오): 포켓 조준 + 스윗스팟 타이밍 자동 플레이 = **~128점**,
  랜덤 플레이 = ~55점. 스윗스팟이 실제로 점수를 끌어올림을 확인.
