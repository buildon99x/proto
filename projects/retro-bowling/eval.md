# Evaluation — Retro Bowling

## 자동 검증
- `pnpm --filter retro-bowling build` — 타입체크(tsc) + Vite 빌드 무오류.
- `pnpm playtest --project retro-bowling` — 헤드리스 크롬으로 타이틀→10프레임→
  게임오버까지 자동 플레이. page/console 에러 0, 스크린샷 8장 캡처.
  시나리오: `tests/e2e/scenario.mjs`.
- 스코어링 단위 검증(수동 스크립트로 확인한 케이스):
  - 퍼펙트 게임(12스트라이크) = 300
  - 올 거터 = 0, 올 9핀 오픈 = 90
  - 올 스페어(5/5…)+마지막 5 = 150
  - 위키 표준 예시 = 133
  - 10프레임 X X X 단독 = 30
  - 보너스 미확정 스트라이크 프레임 누적 = null

## 수동 체크리스트
- [ ] 타이틀에서 시작되고 조준 가이드가 좌우로 움직인다.
- [ ] 파워/스핀 미터가 왕복하고 SPACE 로 고정된다.
- [ ] 스핀 방향에 따라 공이 좌/우로 휘어진다(훅).
- [ ] 포켓에 넣으면 핀이 연쇄로 쓰러지고 스트라이크가 난다.
- [ ] 거터로 빠지면 GUTTER 배너, 핀 0개.
- [ ] 상단 스코어시트가 표준 볼링 규칙대로 집계된다(스페어=다음 1구 보너스 등).
- [ ] 10프레임 스트라이크/스페어 시 보너스 공이 주어진다.
- [ ] 게임오버에서 최종 점수·랭크·하이스코어가 표시되고 재시작된다.
- [ ] 모바일 터치 버튼(◀ ● ▶)과 캔버스 탭으로 조작된다.
- [ ] 도트 확대가 선명하고(픽셀), 스캔라인/팔레트가 레트로하다.

## 알려진 한계
- 물리는 아케이드 감성 위주의 근사(실제 볼링 궤적과 다름).
- 스플릿/특정 핀 배치의 세밀한 재현보다 재미와 반응성 우선.
