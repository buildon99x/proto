# Spec — Retro Bowling

## 개요
캔버스 기반 도트 그래픽 볼링 게임. 내부 해상도 256x384(세로)에 그린 뒤 CSS
`image-rendering: pixelated` 로 확대해 도트 질감을 유지한다. React 는 얇은 셸이고
게임 로직/렌더는 순수 TypeScript 모듈.

## 화면(Screens) / 상태(Phases)
게임은 단일 캔버스에서 상태 머신으로 전환한다(`engine.ts`의 `Phase`).

- `title` — 타이틀. SPACE/탭 → 게임 시작.
- `aim` — `< >` 로 조준 x 이동, 조준 가이드 점선 표시. SPACE → `power`.
- `power` — 파워 미터 0→1→0 왕복. SPACE → 파워 고정, `spin`.
- `spin` — 스핀 미터 -1→+1 왕복(중앙 0). SPACE → 스핀 고정 + 투구, `rolling`.
- `rolling` — 볼/핀 물리 시뮬레이션. 정지하면 정산.
- `result` — STRIKE/SPARE/GUTTER/MISS/N PINS 배너 표시 후 다음 공.
- `gameover` — 최종 점수·랭크·하이스코어·미니 스코어시트. SPACE/탭 → 타이틀.

## 조작
- 키보드: `←/→`(또는 A/D) 조준, `Space`/`Enter`/`↑` 액션, `M` 음소거.
- 터치/마우스: 화면 하단 `◀ ● ▶` 버튼, 캔버스 탭 = 액션.

## 물리(engine.ts)
- 레인 좌표계: x∈[0,50](거터 5+5, 플레이 5..45), y∈[0,240](파울라인→핀덱).
- 볼: 파워→초기 y속도(96~178), 스핀→구르는 동안 곡선 가속(느릴수록 훅 증가).
  마찰 감쇠, 거터 판정, 볼-핀 원충돌(질량 6:1).
- 핀 10개: 표준 삼각 배치. 핀-핀 탄성 충돌로 연쇄 반응·스플릿 자연 발생.
  홈에서 2.4u 이상 벗어나면 쓰러진 것으로 판정, 정산 시 치워짐.
- 고정 스텝(1/240s) 적분으로 관통 방지.

## 점수(scoring.ts)
- 표준 10프레임. 1~9프레임 스트라이크=[10], 스페어=[a,10-a], 오픈=[a,b].
- 10프레임은 스트라이크/스페어 시 3구.
- 보너스 미확정 프레임은 누적 표시 보류(null). 랭크: 점수 구간별 ROOKIE~PERFECT.

## 렌더(render.ts)
- 원근 투영(사다리꼴 레인, 널판 줄무늬, 타깃 애로우, 파울라인).
- 핀은 먼 것부터 그려 겹침 처리, 볼은 그림자+하이라이트+스핀 점.
- 상단 10프레임 스코어보드(현재 프레임 강조), 하단 파워/스핀 미터 + 프롬프트.
- 자체 5x7 비트맵 폰트(`pixelfont.ts`)로 모든 텍스트 렌더.

## 사운드(audio.ts)
- WebAudio 즉석 합성 칩튠: 투구/핀타격/스트라이크/스페어/거터/미스/시작/게임오버.

## 영속화
- `localStorage`(`retro-bowling/high/v1`)에 하이스코어 저장.

## 비목표(Non-goals)
- 멀티플레이어, 온라인 랭킹, 3D, 계정.

## 난이도 설계: 쉬운 조작 / 어려운 완전정복 (v0.2)
"실수해도 핀은 쓰러지지만, 스트라이크는 정교함이 필요하다."

- **바닥 낮추기(접근성)**
  - 라이브 궤도 프리뷰: 조준/파워/스핀 단계에서 예상 경로를 실시간 표시(스핀 단계엔
    오실레이터 값에 따라 훅 아크가 휘어짐). `predictPath()`가 실제 물리와 동일 공식.
  - 추천 조준 마커(POCKET): 겨냥 지점을 레인에 표시.
  - 높은 평균 핀(직구 평균 ~8.7): 대충 굴려도 7~9핀은 쓰러짐.
- **천장 높이기(마스터리)**
  - 포켓 강제: 정중앙 직구는 스트라이크 불가(볼이 정면 충돌 시 감속·편향). 헤드핀 우측
    + 좌향 훅으로 1-3 포켓에 각을 넣어야 스트라이크가 응집(`POCKET` 기준값).
  - 스윗스팟 미터: 파워(≈0.73)·스핀(≈-0.1)의 PERFECT/GOOD 존(`SWEET`). 오실레이팅
    마커를 존에 정확히 멈춰야 이상적 라인. 결정론적이라 마스터리가 재현 가능하되,
    미터 타이밍이 실행 난이도를 만든다.
  - 스코어링 복리: 스트라이크는 다음 2구 보너스까지 최대 30점 → 연속 스트라이크가
    점수를 폭증시키고, 한 번의 실수가 스트링을 끊는다. 300점은 12연속 포켓+정타 요구.

## 물리 튜닝 (`TUNE`, `POCKET`, `SWEET`)
- `TUNE`: 볼 속도/마찰/스핀/복원계수/질량/쓰러짐 임계. 헤드리스 스트라이크맵으로 튜닝.
- `POCKET`: 추천 조준·파워·스핀(스트라이크 라인). 프리뷰·마커·미터 존의 기준.
- `SWEET`: 파워/스핀의 PERFECT·GOOD 구간(-1..1 및 0..1 공간).
