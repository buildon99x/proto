# kb-extract — 세션 지식 추출

현재 대화에서 수행한 작업을 되짚어 **ADR**(내린 결정)과 **LRN**(겪은 시행착오)을 추출하고 `.context/kb/`에 저장한다.

## 절차

1. **기존 KB 파악**
   - `.context/kb/README.md`와 `adr/`, `lrn/` 파일 목록을 읽어 마지막 번호와 기존 주제를 파악한다.
   - 디렉터리가 없으면 README.md(구조·규칙·빈 인덱스 포함)와 함께 생성한다.

2. **세션 회고 — 후보 추출**
   - **ADR 후보:** 복수의 대안 중 하나를 *이유를 갖고* 선택한 지점. (도구/라이브러리/접근법 선택, 데이터 포맷, 아키텍처 원칙, 스코프 컷). 사소한 구현 디테일은 제외 — "다음 사람이 왜?라고 물을 결정"만.
   - **LRN 후보:** 처음 시도가 틀렸고 원인을 찾아 고친 지점. (잘못된 가정, 버그, 차단/우회, 놓친 변형 케이스). 한 번에 성공한 일은 LRN이 아니다.
   - 각 후보에 대해 자문: 이 기록이 **다음 세션의 행동을 바꾸는가?** 아니라면 버린다.

3. **중복 검사**
   - 기존 ADR/LRN과 같은 주제면 새 파일 대신 기존 파일에 추가·갱신한다 (ADR이 뒤집히면 새 ADR 작성 + 옛 ADR 상태를 "대체됨"으로).

4. **작성 — 형식 준수**
   - 파일명: `ADR-NNNN-slug.md` / `LRN-NNNN-slug.md` (번호는 디렉터리별 이어서 증가, 4자리, slug는 영문 kebab-case).
   - **ADR 형식:**
     ```markdown
     # ADR-NNNN: <결정을 한 문장으로>
     - **상태:** 채택|대체됨(→ADR-XXXX) (YYYY-MM-DD)
     - **관련 작업:** <무슨 작업이었나>
     ## 맥락        <검토한 대안들과 제약>
     ## 결정        <선택과 이유>
     ## 결과        <얻은 것 / 감수한 트레이드오프>
     ```
   - **LRN 형식:**
     ```markdown
     # LRN-NNNN: <교훈을 한 문장으로>
     - **날짜:** YYYY-MM-DD · **작업:** <맥락>
     ## 상황 / 시행착오   <무엇을 시도했고 어떻게 틀렸나 — 증상 포함>
     ## 원인              <실제 원인>
     ## 해결              <고친 방법>
     ## 교훈              <번호 목록 — 다음에 행동을 바꿀 문장으로>
     ```
   - 구체적 수치·에러 메시지·파일 경로를 포함해 검색 가능하게 쓴다. 관련 ADR↔LRN은 상대 링크로 상호 참조.

5. **인덱스 갱신**
   - `.context/kb/README.md`의 ADR/LRN 인덱스 표에 새 항목을 추가한다.

6. **보고**
   - 생성/갱신한 파일 목록과 각 항목 한 줄 요약을 사용자에게 보고한다. 커밋은 사용자가 요청했거나 진행 중인 작업 흐름에 커밋이 포함된 경우에만.

## 품질 기준

- ADR 5±3개, LRN 5±3개가 일반적 — 양보다 재사용성. 얇은 항목 여러 개보다 밀도 있는 항목 소수가 낫다.
- 교훈은 일반화하되 근거가 된 구체 사례를 남긴다 ("X하라" + "이번에 Y에서 이렇게 데였다").
- 자기비판을 검열하지 않는다 — 틀린 가정·낭비된 시도가 LRN의 원료다.
